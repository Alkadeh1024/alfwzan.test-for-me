موقع ثابت (Static) كامل بـ HTML/CSS/JS فقط، لا يحتاج إلى أي خادم. مناسب للنشر على **GitHub Pages** مباشرة.

## النشر على GitHub Pages

1. أنشئ مستودع جديد على GitHub (مثلا `alfwzan-site`).
2. ارفع كل محتويات هذا المجلد إلى الفرع `main`.
3. من إعدادات المستودع: **Settings → Pages → Source: Deploy from branch → main / (root)**.
4. سيظهر رابط الموقع خلال دقيقة، مثل: `https://USERNAME.github.io/alfwzan-site/`.

> الملف `.nojekyll` موجود ليمنع Jekyll من تجاهل أي ملفات.

## النشر السريع من داخل لوحة المشرف (موصى به)

1. سجّل دخول كمشرف من `admin.html`.
2. اضغط **«إعدادات النشر»** بأعلى اللوحة وأدخل:
   - اسم المستخدم على GitHub (مثل: `Alkadeh1024`)
   - اسم المستودع (مثل: `alfwzan`)
   - اسم الفرع: `main`
   - **Personal Access Token (PAT)** — أنشئه من:
     `GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens` ثم امنحه صلاحية **Contents: Read and write** على مستودع الموقع فقط.
3. اضغط **حفظ**. ثم في أي وقت أضف صور/فيديوهات/طلاب/أنشطة، واضغط **«نشر التغييرات»** ليُرفع كل شيء تلقائيا إلى مستودعك على GitHub:
   - كل ملف وسائط يُرفع منفردا في مجلد `media/{id}.{ext}` (يدعم فيديوهات كبيرة عبر IndexedDB).
   - يُحدَّث `data/site.json` بكل البيانات والروابط النهائية.
4. الزوار يفتحون الموقع فيقرأ تلقائيا أحدث `data/site.json` من GitHub Pages — تظهر التعديلات للجميع فوراً.

> زر **«جلب من GitHub»** مفيد لو فتحت اللوحة من جهاز جديد لاستعادة آخر بيانات منشورة.

## ملاحظات

- الوسائط (الصور والفيديوهات) تُخزَّن محلياً في **IndexedDB** (يدعم ملفات بأحجام GB دون تعليق المتصفح).
- التحضير وتقدير الحفظ من حساب الشيخ يُحفظ محلياً ولا يُنشر تلقائياً (يخص حسابه فقط).
- الشعار: ضع ملف باسم `alfwzan.jpeg` داخل `pectirs/` (يظهر تلقائياً، أو يظهر بديل لو غير موجود).
- الموقع كله RTL بخط Aref Ruqaa للعناوين وخلفية بحروف عربية.
